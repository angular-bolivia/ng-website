## Findings considered and rejected

- **`npm audit` "6 high severity" advisories**: all trace to the esbuild/vite
  **dev-server** advisory. This affects only the local dev server, not the static
  HTML deployed to Firebase Hosting nor the build artifacts. Remediation would
  require breaking downgrades for zero production benefit. Not worth doing.
- **Hardcoded GA measurement ID (`G-7K9VWR2PGF`)**: a client-side GA ID is public
  by design and ships in the HTML intentionally — it is not a secret. Moving it
  to an env var adds build config for no security benefit.
- **Image optimization (PNG→WebP, `astro:assets`)**: the components are Angular,
  not Astro, so Astro's `<Image>` pipeline doesn't apply cleanly, and current
  payloads are modest (largest is the 205 KB `meta-icon.png`, which isn't even
  rendered on-page). Low leverage; revisit only if Core Web Vitals regress.
